# gen-slides

Generate publication-quality, fully editable .pptx presentations from academic papers, reports, or topic descriptions.

## Input

The user provides:
1. **Source material**: one or more papers (PDF/LaTeX/Markdown), a thesis project, or a topic description
2. **Presentation type** (if not specified, ask):
   - `seminar` — lab meeting / journal club (8-12 slides)
   - `proposal` — research proposal / 开题 (12-15 slides)
   - `midterm` — mid-term progress report / 中期 (15-20 slides)
   - `defense` — thesis defense / 答辩 (18-25 slides)
   - `group-project` — course project presentation (8-15 slides)

Optional arguments:
- `--logo <path>`: logo image for top-right corner of each slide
- `--colors <primary_hex> <accent_hex>`: brand colors (default: #1F4E79 primary, #2E75B6 accent)
- `--font <name>`: font family (default: system sans-serif appropriate for OS)
- `--lang en|zh`: language for slide titles and structure (default: match source material)
- `--output <path>`: output file path (default: `./<title>.pptx`)

## Process

### Phase 1: Analyze Source Material

Read all provided documents and extract:
- **Title** and **authors**
- **Abstract/summary** (for overview slide)
- **Section structure** (maps to slide sections)
- **Key figures** and their captions (for figure slides)
- **Tables** with important results (for results slides)
- **Core method/contribution** (for method slides)
- **Experimental results** — numbers, comparisons (for results slides)
- **Conclusions and future work**

If source is a LaTeX project: read .tex files directly (richer structure than PDF).
If source is a PDF: read with the Read tool and extract text + note figure locations.

### Phase 2: Plan Slide Structure

Based on presentation type, plan the slide deck:

**defense (18-25 slides):**
1. Title slide (title, author, advisor, institution, date)
2. Outline
3-4. Background & motivation (2 slides)
5-6. Related work (1-2 slides)
7-12. Method (4-6 slides: architecture overview, key modules, formulas)
13-16. Experiments (3-4 slides: setup, quantitative results, visual comparisons)
17-18. Ablation study (1-2 slides, if applicable)
19. Conclusion
20. Future work
21. Thank you / Q&A

**midterm (15-20 slides):**
1. Title slide
2. Outline
3-4. Research background (2 slides)
5. Research objectives
6-7. Proposed method overview (2 slides)
8-12. Current progress & preliminary results (4-5 slides)
13-14. Problems encountered & solutions (1-2 slides)
15. Next steps & timeline
16. References (if needed)

**seminar (8-12 slides):**
1. Title slide (paper title + "presented by")
2. Motivation / problem
3-4. Method core idea (2-3 slides)
5-6. Key results (2 slides)
7. Discussion / limitations
8. Takeaways

**proposal (12-15 slides):**
1. Title slide
2. Research background
3-4. Literature review (2 slides)
5. Research gap & motivation
6-7. Proposed approach (2-3 slides)
8. Expected contributions
9-10. Preliminary work (if any)
11. Timeline / milestones
12. References

**group-project (8-15 slides):**
1. Title slide (project name + team members)
2. Problem statement
3-4. Our approach (2-3 slides)
5-6. Implementation / demo (2-3 slides)
7. Results / evaluation
8. Conclusion & future work

### Phase 3: Generate python-pptx Script

Write a self-contained Python script that:
1. Creates a presentation using python-pptx
2. Applies theme colors, fonts, and layout constants
3. Builds each slide with proper layout:
   - **Title slides**: centered title + subtitle, colored background
   - **Content slides**: colored title bar + bullet points
   - **Figure slides**: title + centered image with caption
   - **Two-column slides**: title + left/right content (for comparisons)
   - **Section divider slides**: large centered text on accent background
4. Adds speaker notes to each slide with talking points
5. Embeds figures from source material (copies to working directory if needed)
6. If `--logo` provided: places logo at top-right of every content slide

```python

"""Generated by academic-arsenal/gen-slides"""

PRIMARY = RGBColor(0x1F, 0x4E, 0x79)

LIGHT_TEXT = RGBColor(0xFF, 0xFF, 0xFF)
DARK_TEXT = RGBColor(0x1A, 0x1A, 0x1A)

def get_font():
    s = platform.system()
    if s == "Darwin": return "PingFang SC"
    elif s == "Windows": return "Microsoft YaHei"
    return "Noto Sans CJK SC"

FONT = get_font()
SLIDE_WIDTH = Inches(13.333)
SLIDE_HEIGHT = Inches(7.5)

def section_slide(prs, title): ...

# --- Build Presentation ---
prs = Presentation()

prs.slide_height = SLIDE_HEIGHT

# [slides built here, one function call per slide]

prs.save("output.pptx")
print(f"Generated: output.pptx ({len(prs.slides)} slides)")
```

**Slide design rules:**
- Title bar: full-width rectangle at top with PRIMARY color, white title text
- Body text: 16pt, dark color, left-aligned with proper margins
- Figures: centered, maximum 70% slide width, with caption below
- Never more than 6 bullet points per slide
- Never more than ~30 words per bullet
- Speaker notes: what to SAY, not what's on the slide

### Phase 4: Run Script

```bash
# Ensure python-pptx is available
python3 -c "import pptx" 2>/dev/null || pip install python-pptx -i https://pypi.tuna.tsinghua.edu.cn/simple

# Run the generated script
python3 build_slides.py
```

### Phase 5: Quality Check

After generating:
1. Verify file exists and report: slide count, file path, file size
2. If any referenced figure file is missing: warn the user
3. If any figure appears low-resolution or is a placeholder: offer to redraw

### Phase 6: Optional Diagram Redraw

If unclear figures detected:
- Inform user: "Figure X appears low-quality. Want me to redraw it using gen-diagram?"
- If yes: invoke gen-diagram skill with description of what the figure should show
- Replace the blurry image with the new diagram export (PNG)

## Important Notes

- Output MUST be truly editable — real text frames, not images of text
- The python-pptx script is saved alongside the .pptx (user can re-run with edits)
- Use Tsinghua PyPI mirror for pip install (faster in China)
- Adapt font choice to OS automatically (PingFang SC / Microsoft YaHei / Noto Sans)
- Figures must be large and legible — if a figure would be smaller than 3 inches wide, give it a full slide
- For Chinese presentations: use Chinese section headers and bullet text
- For English presentations: use English throughout
